# Online Judge 系统 - 数据库选择方案

## 1. 数据库选择策略

### 1.1 开发环境 vs 生产环境

我们采用**分环境数据库策略**，平衡开发便捷性和生产性能：

```
开发环境：SQLite（本地文件数据库，零配置）
    ↓ 开发完成
测试环境：PostgreSQL（Docker 容器）
    ↓ 测试通过
生产环境：PostgreSQL（独立服务器/云数据库）
```

### 1.2 为什么这样选择？

**开发环境使用 SQLite：**
- ✅ **零配置**：无需安装数据库服务
- ✅ **快速启动**：克隆项目即可运行
- ✅ **轻量级**：单文件数据库，易于备份和重置
- ✅ **跨平台**：macOS/Linux/Windows 都支持
- ✅ **适合单人开发**：无需担心数据库连接问题

**生产环境使用 PostgreSQL：**
- ✅ **功能强大**：支持 JSONB、全文搜索、复杂查询
- ✅ **高性能**：支持高并发、连接池
- ✅ **数据完整性**：严格的事务支持
- ✅ **可扩展**：支持读写分离、主从复制
- ✅ **生态成熟**：工具链完善

## 2. 数据库部署方案

### 2.1 开发环境（推荐）

**方案 A：纯 SQLite（最简单）**

```yaml
优点:
  - 零配置，开箱即用
  - 无需 Docker
  - 适合快速原型开发
  
缺点:
  - 不支持某些高级特性
  - 并发性能有限
  
适用场景:
  - 个人开发
  - 快速验证功能
  - 学习和实验
```

**配置示例：**

```python
# backend/.env
DATABASE_URL=sqlite+aiosqlite:///./oj.db
```

**方案 B：SQLite + Redis（推荐）**

```yaml
SQLite: 主数据库（本地文件）
Redis: 缓存和队列（Docker 容器）

优点:
  - 数据库零配置
  - Redis 提供缓存和队列功能
  - 接近生产环境
  
缺点:
  - 需要 Docker
  
适用场景:
  - 日常开发
  - 需要测试缓存功能
```

**启动方式：**

```bash
# 只启动 Redis
docker-compose up -d redis

# 应用使用 SQLite
python main.py
```

**方案 C：完整 Docker 环境**

```yaml
PostgreSQL: Docker 容器
Redis: Docker 容器

优点:
  - 完全模拟生产环境
  - 测试数据库特性
  
缺点:
  - 需要 Docker
  - 资源占用较多
  
适用场景:
  - 测试环境
  - 集成测试
  - 团队协作
```

### 2.2 生产环境

**方案 A：Docker Compose（单机部署）**

```yaml
适用场景: 小型项目、个人项目
配置: docker-compose.yml
优点: 简单、易维护
缺点: 单点故障
```

**方案 B：独立服务器**

```yaml
适用场景: 中型项目
配置: 独立的 PostgreSQL 服务器
优点: 性能好、可优化
缺点: 需要运维
```

**方案 C：云数据库**

```yaml
适用场景: 大型项目、企业项目
服务商: AWS RDS、阿里云 RDS、腾讯云 CDB
优点: 高可用、自动备份、易扩展
缺点: 成本较高
```

## 3. 推荐的开发流程

### 3.1 初始开发阶段（当前）

```bash

# 2. 安装依赖（使用 uv）
```

```bash
# 2. 配置环境变量

# 3. 启动应用
```

### 4.2 完整配置（PostgreSQL + Redis）

```yaml

  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: oj
      POSTGRES_USER: oj
      POSTGRES_PASSWORD: ${DB_PASSWORD:-oj_password}

      - "6379:6379"
```

## 5. SQLAlchemy 配置（支持多数据库）

### 5.1 数据库配置文件

```python
# backend/app/core/database.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker, declarative_base
from app.core.config import settings

# 根据配置自动选择数据库
engine = create_async_engine(
    settings.DATABASE_URL,
    echo=settings.DEBUG,
    future=True,
    # SQLite 特殊配置
    connect_args={"check_same_thread": False} if "sqlite" in settings.DATABASE_URL else {}
)

AsyncSessionLocal = sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)

Base = declarative_base()

async def get_db():
    """依赖注入：获取数据库会话"""
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
        finally:
            await session.close()
```

### 5.2 环境配置

```python
# backend/app/core/config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    # 数据库配置
    DATABASE_URL: str = "sqlite+aiosqlite:///./oj.db"
    
    # Redis 配置
    REDIS_URL: str = "redis://localhost:6379/0"
    
    # 应用配置
    DEBUG: bool = True
    
    class Config:
        env_file = ".env"

settings = Settings()
```

### 5.3 多环境配置示例


## 7. 数据库兼容性注意事项

### 7.1 SQLite 限制

```python
# ❌ SQLite 不支持
ALTER TABLE users ADD COLUMN age INTEGER DEFAULT 18;  # 某些 ALTER 操作
CREATE INDEX CONCURRENTLY idx_name ON users(name);    # 并发索引

# ✅ 使用兼容写法
# 在迁移中检查数据库类型
if engine.dialect.name == 'sqlite':
    # SQLite 特殊处理
else:
    # PostgreSQL 处理
```

### 7.2 推荐做法

```python
# 使用 SQLAlchemy ORM，自动处理差异
from sqlalchemy import Column, Integer, String, DateTime, JSON
from sqlalchemy.dialects.postgresql import JSONB

class Problem(Base):
    __tablename__ = "problems"
    
    id = Column(Integer, primary_key=True)
    title = Column(String(200), nullable=False)
    
    # 自动适配：PostgreSQL 用 JSONB，SQLite 用 JSON
    metadata = Column(JSON, nullable=True)
```

## 8. 性能对比

### 8.1 开发环境性能

| 操作 | SQLite | PostgreSQL (Docker) | 差异 |
|------|--------|---------------------|------|
| 启动时间 | 即时 | 3-5秒 | SQLite 更快 |
| 简单查询 | 1ms | 2ms | 相近 |
| 复杂查询 | 10ms | 8ms | PostgreSQL 稍快 |
| 并发写入 | 慢 | 快 | PostgreSQL 显著优势 |
| 内存占用 | 10MB | 100MB | SQLite 更省 |

### 8.2 生产环境性能

PostgreSQL 在生产环境下性能远超 SQLite：
- 支持数千并发连接
- 支持复杂查询优化
- 支持分区表、物化视图
- 支持读写分离

## 10. 迁移路径

### 10.1 从 SQLite 迁移到 PostgreSQL

```bash
# 1. 导出 SQLite 数据
sqlite3 oj.db .dump > dump.sql

# 2. 转换 SQL（可能需要手动调整）
# SQLite 和 PostgreSQL 语法有差异

# 3. 导入到 PostgreSQL
psql -U oj -d oj < dump.sql

# 或使用工具
pip install pgloader
pgloader oj.db postgresql://oj:password@localhost/oj
```

### 10.2 无缝切换

由于使用 SQLAlchemy ORM，切换数据库只需：

```bash
# 1. 修改环境变量
DATABASE_URL=postgresql+asyncpg://...

# 2. 运行迁移
alembic upgrade head

# 3. 重启应用
# 无需修改代码！
```

```
    ↓
```

- ✅ 开发零配置，快速启动

- ✅ 生产环境使用最佳方案
