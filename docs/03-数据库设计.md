# Online Judge - 数据库设计

> 最后更新：2026-03-17

## 1. 数据库选型

**当前（开发）：** SQLite（`src/backend/oj.db`）
**规划（生产）：** PostgreSQL
- 支持复杂查询和事务
- 性能优秀
- 支持 JSON 字段

**缓存数据库：** Redis
- 高性能键值存储
- 支持多种数据结构
- 用于缓存和会话管理

## 2. 数据表设计

ORM 框架：SQLAlchemy 2.0（async），迁移工具：Alembic。

### 2.1 用户表 (users)

```python
# app/models/user.py
class User(Base):
    __tablename__ = "users"
    id: int (PK)
    username: str(50)  UNIQUE NOT NULL
    email: str(100)    UNIQUE NOT NULL
    hashed_password: str(255) NOT NULL
    role: Enum("user","admin","super_admin")  DEFAULT "user"
    is_active: bool  DEFAULT True
    avatar: str(255) NULLABLE
    created_at: datetime  DEFAULT now()
    updated_at: datetime  NULLABLE
```

### 2.2 题目表 (problems) + 测试用例表 (test_cases)

```python
# app/models/problem.py
class Problem(Base):
    __tablename__ = "problems"
    id: int (PK)
    title: str(200)  NOT NULL  INDEX
    description: Text  NOT NULL        # Markdown
    difficulty: Enum("easy","medium","hard")  DEFAULT "easy"  INDEX
    input_format: Text  NULLABLE
    output_format: Text  NULLABLE
    constraints: Text  NULLABLE
    time_limit: int  DEFAULT 1000      # 毫秒
    memory_limit: int  DEFAULT 256     # MB
    tags: JSON  NULLABLE               # 字符串数组
    source: str(100)  NULLABLE         # 题目来源
    submission_count: int  DEFAULT 0
    accepted_count: int  DEFAULT 0
    is_public: bool  DEFAULT True  INDEX
    created_at: datetime
    updated_at: datetime  NULLABLE

class TestCase(Base):
    __tablename__ = "test_cases"
    id: int (PK)
    problem_id: int  FK(problems.id CASCADE)  INDEX
    input: Text  NOT NULL
    output: Text  NOT NULL
    is_sample: bool  DEFAULT False     # 样例会展示给用户
    score: int  DEFAULT 10             # 分数权重（部分分）
    created_at: datetime
```

### 2.3 提交记录表 (submissions)

```python
# app/models/submission.py
class SubmissionStatus(Enum):
    PENDING = "pending"
    JUDGING = "judging"
    ACCEPTED = "accepted"
    WRONG_ANSWER = "wrong_answer"
    TIME_LIMIT_EXCEEDED = "time_limit_exceeded"
    MEMORY_LIMIT_EXCEEDED = "memory_limit_exceeded"
    RUNTIME_ERROR = "runtime_error"
    COMPILE_ERROR = "compile_error"
    SYSTEM_ERROR = "system_error"

class Submission(Base):
    __tablename__ = "submissions"
    id: int (PK)
    user_id: int  FK(users.id CASCADE)  INDEX
    problem_id: int  FK(problems.id CASCADE)  INDEX
    language: Enum("python","cpp","java","go","javascript","c")
    code: Text  NOT NULL
    status: SubmissionStatus  DEFAULT "pending"  INDEX
    score: int  NULLABLE               # 0-100
    time_used: int  NULLABLE           # 毫秒
    memory_used: int  NULLABLE         # KB
    error_message: Text  NULLABLE
    test_cases_result: Text  NULLABLE  # JSON 字符串，每个测试点结果
    submitted_at: datetime
    judged_at: datetime  NULLABLE
```

### 2.4 竞赛相关表

```python
# app/models/contest.py
class Contest(Base):
    __tablename__ = "contests"
    id: int (PK)
    title: str(200)  NOT NULL  INDEX
    description: Text  NULLABLE
    start_time: datetime  NOT NULL  INDEX
    end_time: datetime  NOT NULL  INDEX
    duration: int  NOT NULL            # 分钟
    contest_type: Enum("public","private")  DEFAULT "public"
    rule_type: Enum("acm","oi")  DEFAULT "acm"
    password: str(100)  NULLABLE
    is_visible: bool  DEFAULT True
    real_time_rank: bool  DEFAULT True
    freeze_time: int  NULLABLE         # 封榜（结束前 N 分钟）
    participant_count: int  DEFAULT 0
    submission_count: int  DEFAULT 0
    created_by: int  NOT NULL  INDEX
    created_at / updated_at: datetime

class ContestProblem(Base):
    __tablename__ = "contest_problems"
    id: int (PK)
    contest_id: int  FK(contests.id)  INDEX
    problem_id: int  FK(problems.id)  INDEX
    display_id: str(10)  NOT NULL      # A, B, C...
    score: int  DEFAULT 100
    submission_count / accepted_count: int

class ContestParticipant(Base):
    __tablename__ = "contest_participants"
    id: int (PK)
    contest_id: int  FK(contests.id)  INDEX
    user_id: int  FK(users.id)  INDEX
    is_registered: bool  DEFAULT True
    total_score: int  DEFAULT 0        # OI 总分
    solved_count: int  DEFAULT 0       # ACM 通过题数
    total_time: int  DEFAULT 0         # ACM 罚时（分钟）
    submission_count: int  DEFAULT 0
    registered_at: datetime
    last_submission_at: datetime  NULLABLE
```

## 3. 表关系图

```
users
  └─ submissions (1:N)

problems
  ├─ test_cases (1:N)
  └─ submissions (1:N)

contests
  ├─ contest_problems (1:N) → problems
  └─ contest_participants (1:N) → users
```

## 4. Redis 用途（当前实际）

Redis 当前仅用于 **Celery 消息队列**：
- broker：`redis://localhost:6379/0`
- backend：`redis://localhost:6379/0`
- 队列名：`judge`（判题任务）、`celery`（默认）

> 缓存功能尚未实现，是后续优化方向。

## 5. 迁移管理

使用 Alembic 管理数据库变更：

```bash
cd src/backend

# 生成迁移文件
alembic revision --autogenerate -m "description"

# 执行迁移
alembic upgrade head

# 查看历史
alembic history
```

迁移文件位于 `src/backend/alembic/versions/`。
