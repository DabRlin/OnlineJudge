# Online Judge 系统 - 数据库设计

## 1. 数据库选型

**主数据库：** PostgreSQL
- 支持复杂查询和事务
- 性能优秀
- 支持 JSON 字段

**缓存数据库：** Redis
- 高性能键值存储
- 支持多种数据结构
- 用于缓存和会话管理

## 2. 数据表设计

### 2.1 用户表 (users)

```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),
    avatar_url VARCHAR(255),
    role VARCHAR(20) DEFAULT 'user', -- user, admin, problem_setter
    school VARCHAR(100),
    major VARCHAR(100),
    student_id VARCHAR(50),
    bio TEXT,
    
    -- 统计字段
    solved_count INT DEFAULT 0,
    submission_count INT DEFAULT 0,
    accepted_count INT DEFAULT 0,
    rating INT DEFAULT 1500,
    
    -- 状态字段
    is_active BOOLEAN DEFAULT true,
    is_verified BOOLEAN DEFAULT false,
    
    -- 时间字段
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_rating ON users(rating DESC);
```

### 2.2 题目表 (problems)

```sql
CREATE TABLE problems (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    input_description TEXT,
    output_description TEXT,
    
    -- 限制条件
    time_limit INT NOT NULL, -- 毫秒
    memory_limit INT NOT NULL, -- MB
    stack_limit INT DEFAULT 128, -- MB
    
    -- 难度和分类
    difficulty VARCHAR(20), -- easy, medium, hard
    source VARCHAR(100), -- 题目来源
    
    -- 样例
    sample_input TEXT,
    sample_output TEXT,
    hint TEXT,
    
    -- 统计信息
    submission_count INT DEFAULT 0,
    accepted_count INT DEFAULT 0,
    
    -- 状态
    is_public BOOLEAN DEFAULT false,
    is_special_judge BOOLEAN DEFAULT false,
    
    -- 创建者
    created_by BIGINT REFERENCES users(id),
    
    -- 时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_problems_difficulty ON problems(difficulty);
CREATE INDEX idx_problems_is_public ON problems(is_public);
CREATE INDEX idx_problems_created_at ON problems(created_at DESC);
```

### 2.3 题目标签表 (tags)

```sql
CREATE TABLE tags (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT,
    color VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE problem_tags (
    problem_id BIGINT REFERENCES problems(id) ON DELETE CASCADE,
    tag_id INT REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (problem_id, tag_id)
);

CREATE INDEX idx_problem_tags_problem ON problem_tags(problem_id);
CREATE INDEX idx_problem_tags_tag ON problem_tags(tag_id);
```

### 2.4 测试数据表 (test_cases)

```sql
CREATE TABLE test_cases (
    id BIGSERIAL PRIMARY KEY,
    problem_id BIGINT REFERENCES problems(id) ON DELETE CASCADE,
    
    -- 测试数据
    input_file VARCHAR(255), -- 文件路径或内联数据
    output_file VARCHAR(255),
    input_size BIGINT, -- 字节
    output_size BIGINT,
    
    -- 配置
    score INT DEFAULT 0, -- 用于 OI 模式
    is_sample BOOLEAN DEFAULT false,
    sort_order INT DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_test_cases_problem ON test_cases(problem_id);
```

### 2.5 提交记录表 (submissions)

```sql
CREATE TABLE submissions (
    id BIGSERIAL PRIMARY KEY,
    problem_id BIGINT REFERENCES problems(id),
    user_id BIGINT REFERENCES users(id),
    contest_id BIGINT REFERENCES contests(id), -- 可为空
    
    -- 代码信息
    language VARCHAR(20) NOT NULL, -- cpp, python, java, go, etc.
    code TEXT NOT NULL,
    code_length INT,
    
    -- 判题结果
    status VARCHAR(20) DEFAULT 'pending', -- pending, judging, accepted, wrong_answer, etc.
    score INT DEFAULT 0,
    
    -- 执行信息
    time_used INT, -- 毫秒
    memory_used INT, -- KB
    
    -- 详细结果
    test_cases_passed INT DEFAULT 0,
    test_cases_total INT DEFAULT 0,
    error_message TEXT,
    
    -- IP 和环境
    ip_address VARCHAR(45),
    user_agent TEXT,
    
    -- 是否公开
    is_public BOOLEAN DEFAULT true,
    
    -- 时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    judged_at TIMESTAMP
);

CREATE INDEX idx_submissions_user ON submissions(user_id, created_at DESC);
CREATE INDEX idx_submissions_problem ON submissions(problem_id, created_at DESC);
CREATE INDEX idx_submissions_contest ON submissions(contest_id, created_at DESC);
CREATE INDEX idx_submissions_status ON submissions(status);
CREATE INDEX idx_submissions_created_at ON submissions(created_at DESC);
```

### 2.6 判题详情表 (judge_results)

```sql
CREATE TABLE judge_results (
    id BIGSERIAL PRIMARY KEY,
    submission_id BIGINT REFERENCES submissions(id) ON DELETE CASCADE,
    test_case_id BIGINT REFERENCES test_cases(id),
    
    -- 结果
    status VARCHAR(20), -- AC, WA, TLE, MLE, RE, etc.
    time_used INT, -- 毫秒
    memory_used INT, -- KB
    score INT DEFAULT 0,
    
    -- 错误信息
    error_message TEXT,
    
    -- 顺序
    test_case_order INT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_judge_results_submission ON judge_results(submission_id);
```

### 2.7 竞赛表 (contests)

```sql
CREATE TABLE contests (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    
    -- 时间
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP NOT NULL,
    duration INT, -- 分钟
    
    -- 类型
    type VARCHAR(20) DEFAULT 'acm', -- acm, oi, ioi
    
    -- 权限
    is_public BOOLEAN DEFAULT true,
    password VARCHAR(100), -- 加密后的密码
    
    -- 规则
    rule_type VARCHAR(20) DEFAULT 'acm', -- acm, oi
    
    -- 创建者
    created_by BIGINT REFERENCES users(id),
    
    -- 状态
    status VARCHAR(20) DEFAULT 'upcoming', -- upcoming, running, ended
    
    -- 统计
    participant_count INT DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_contests_start_time ON contests(start_time DESC);
CREATE INDEX idx_contests_status ON contests(status);
```

### 2.8 竞赛题目关联表 (contest_problems)

```sql
CREATE TABLE contest_problems (
    id BIGSERIAL PRIMARY KEY,
    contest_id BIGINT REFERENCES contests(id) ON DELETE CASCADE,
    problem_id BIGINT REFERENCES problems(id),
    
    -- 竞赛中的编号
    problem_index VARCHAR(10), -- A, B, C, ...
    
    -- 分数（OI 模式）
    score INT DEFAULT 100,
    
    -- 顺序
    sort_order INT DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(contest_id, problem_id),
    UNIQUE(contest_id, problem_index)
);

CREATE INDEX idx_contest_problems_contest ON contest_problems(contest_id);
```

### 2.9 竞赛参与者表 (contest_participants)

```sql
CREATE TABLE contest_participants (
    id BIGSERIAL PRIMARY KEY,
    contest_id BIGINT REFERENCES contests(id) ON DELETE CASCADE,
    user_id BIGINT REFERENCES users(id),
    
    -- 排名信息
    rank INT,
    score INT DEFAULT 0,
    total_time INT DEFAULT 0, -- ACM 模式罚时
    
    -- 状态
    is_banned BOOLEAN DEFAULT false,
    
    -- 时间
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(contest_id, user_id)
);

CREATE INDEX idx_contest_participants_contest ON contest_participants(contest_id);
CREATE INDEX idx_contest_participants_user ON contest_participants(user_id);
```

### 2.10 讨论表 (discussions)

```sql
CREATE TABLE discussions (
    id BIGSERIAL PRIMARY KEY,
    problem_id BIGINT REFERENCES problems(id),
    user_id BIGINT REFERENCES users(id),
    
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    
    -- 类型
    type VARCHAR(20) DEFAULT 'general', -- general, solution, question
    
    -- 统计
    view_count INT DEFAULT 0,
    like_count INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    
    -- 状态
    is_pinned BOOLEAN DEFAULT false,
    is_locked BOOLEAN DEFAULT false,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_discussions_problem ON discussions(problem_id);
CREATE INDEX idx_discussions_user ON discussions(user_id);
CREATE INDEX idx_discussions_created_at ON discussions(created_at DESC);
```

### 2.11 评论表 (comments)

```sql
CREATE TABLE comments (
    id BIGSERIAL PRIMARY KEY,
    discussion_id BIGINT REFERENCES discussions(id) ON DELETE CASCADE,
    user_id BIGINT REFERENCES users(id),
    parent_id BIGINT REFERENCES comments(id), -- 用于回复
    
    content TEXT NOT NULL,
    
    -- 统计
    like_count INT DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_comments_discussion ON comments(discussion_id);
CREATE INDEX idx_comments_user ON comments(user_id);
CREATE INDEX idx_comments_parent ON comments(parent_id);
```

### 2.12 用户解题记录表 (user_problem_status)

```sql
CREATE TABLE user_problem_status (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    problem_id BIGINT REFERENCES problems(id),
```

## 3. Redis 缓存设计

### 3.1 缓存键命名规范

```
# 用户会话
session:{session_id} -> user_data (Hash, TTL: 7天)

# 用户信息
user:{user_id} -> user_data (Hash, TTL: 30分钟)

# 题目信息
problem:{problem_id} -> problem_data (Hash, TTL: 10分钟)

# 题目列表
problems:list:{page}:{filter} -> problem_ids (List, TTL: 5分钟)

# 提交状态
submission:{submission_id}:status -> status (String, TTL: 1小时)

# 竞赛排行榜
contest:{contest_id}:ranklist -> ranklist_data (Sorted Set, TTL: 1分钟)

# 判题队列
judge:queue:high -> submission_ids (List, 无TTL)
judge:queue:normal -> submission_ids (List, 无TTL)
judge:queue:low -> submission_ids (List, 无TTL)

# 限流
ratelimit:{user_id}:{action} -> count (String, TTL: 根据限流策略)

# 在线用户
online:users -> user_ids (Set, TTL: 5分钟)
```

### 3.2 缓存更新策略

- **Cache Aside Pattern**: 读取时检查缓存，未命中则查数据库并写入缓存
- **Write Through**: 更新数据时同时更新缓存
- **Cache Invalidation**: 数据变更时删除相关缓存

## 4. 数据库优化策略

### 4.1 索引优化
- 为常用查询字段建立索引
- 使用复合索引优化多条件查询
- 定期分析和优化索引

### 4.2 分区策略
- 提交记录按时间分区（月度分区）
- 判题结果按提交ID分区

### 4.3 归档策略
- 旧竞赛数据归档
- 旧提交记录归档（保留6个月）

### 4.4 查询优化
- 使用连接池
- 避免 N+1 查询
- 使用批量操作
- 适当使用数据库视图

## 5. 数据备份策略

- **全量备份**: 每天凌晨执行
- **增量备份**: 每小时执行
- **备份保留**: 保留最近30天
- **异地备份**: 定期同步到异地存储
