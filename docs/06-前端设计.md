## 3. 核心页面说明

### 3.1 题目详情页（`ProblemDetail`）
- 左侧：题目描述（Markdown + KaTeX 渲染）
- 右侧：Monaco Editor 代码编辑区 + 语言选择器 + 提交按钮
- 提交后实时轮询：每 1.5s 请求一次，终态（非 pending/judging）停止，最多 30 次
- 底部：该题目的提交历史列表

### 3.2 提交详情页（`SubmissionDetail`）
- 代码高亮展示（Monaco Editor 只读模式）
- 每个测试用例通过情况（状态、耗时）
- 编译错误 / 运行错误信息展示

### 3.3 竞赛相关页面
- **Contests**：列表，支持按状态筛选（未开始/进行中/已结束），关键词搜索
- **ContestDetail**：竞赛题目列表，实时排行榜（ACM 罚时 / OI 总分），报名按钮
- **ContestEdit**（管理员）：创建/编辑竞赛，配置题目及分值

### 3.4 题目编辑页（`ProblemEdit`，管理员）
- 题目基本信息表单
- Markdown 描述编辑（实时预览）
- 测试用例管理（增删改查，样例标记）

## 4. 认证状态管理（`authStore.ts`）

```typescript
interface AuthStore {
  user: User | null
  token: string | null
  login: (user: User, token: string) => void
  logout: () => void
}
```

- Token 持久化到 `localStorage`
- Axios 拦截器自动注入 `Authorization: Bearer <token>` 请求头
- 401 响应自动触发 logout

## 5. API 服务封装（`services/api.ts`）

```typescript
const api = axios.create({
  baseURL: "http://localhost:8000/api/v1",
  timeout: 30000,
})

// 请求拦截：注入 Token
api.interceptors.request.use(config => {
  const token = useAuthStore.getState().token
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：401 自动登出
api.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      useAuthStore.getState().logout()
    }
    return Promise.reject(err)
  }
)
```
