## 2. 核心模块设计

### 2.1 用户服务 (User Service)

**职责：**
- 用户注册、登录、认证
- 用户信息管理
- 权限控制
- Session/Token 管理

**技术选型：**
- JWT 认证
- BCrypt 密码加密
- Redis 存储 Session

**接口：**
- POST /api/user/register
- POST /api/user/login
- GET /api/user/profile
- PUT /api/user/profile
- POST /api/user/logout

### 2.2 题目服务 (Problem Service)

**职责：**
- 题目 CRUD
- 题目列表查询、搜索、过滤
- 题目标签管理
- 题目统计信息

**数据结构：**
- 题目基本信息
- 测试数据（输入/输出）
- 时间/内存限制
- 难度、标签

**接口：**
- GET /api/problems
- GET /api/problems/:id
- POST /api/problems (管理员)
- PUT /api/problems/:id (管理员)
- DELETE /api/problems/:id (管理员)

### 2.3 提交服务 (Submission Service)

**职责：**
- 接收代码提交
- 提交记录管理
- 提交状态查询

**接口：**
- POST /api/submissions
- GET /api/submissions/:id
- GET /api/submissions (用户提交历史)

### 2.4 判题调度服务 (Judge Scheduler)

**职责：**
- 接收判题任务
- 任务队列管理
- 负载均衡分发
- 判题结果回调

**工作流程：**
1. 接收提交请求
2. 验证代码和题目
3. 将任务推入消息队列
4. 返回提交 ID
5. 异步等待判题结果
6. 更新数据库

### 2.5 判题工作节点 (Judge Worker)

**职责：**
- 从队列获取任务
- 代码编译
- 沙箱执行
- 结果对比
- 返回判题结果

**判题流程：**
```
1. 获取任务
2. 创建临时工作目录
3. 写入用户代码
4. 编译代码（如需要）
   ├─ 成功 → 继续
   └─ 失败 → 返回 CE (Compilation Error)
5. 对每个测试点：
   ├─ 创建 Docker 容器
   ├─ 设置资源限制（CPU、内存、时间）
   ├─ 执行程序，输入测试数据
   ├─ 捕获输出
   ├─ 对比输出
   │  ├─ 正确 → AC (Accepted)
   │  ├─ 错误 → WA (Wrong Answer)
   │  ├─ 超时 → TLE (Time Limit Exceeded)
   │  ├─ 超内存 → MLE (Memory Limit Exceeded)
   │  └─ 运行错误 → RE (Runtime Error)
   └─ 销毁容器
6. 汇总结果
7. 清理临时文件
8. 返回结果
```

### 2.6 竞赛服务 (Contest Service)

**职责：**
- 竞赛创建和管理
- 竞赛题目关联
- 排行榜计算
- 竞赛权限控制

**接口：**
- GET /api/contests
- GET /api/contests/:id
- POST /api/contests (管理员)
- GET /api/contests/:id/ranklist

## 3. 数据流设计

### 3.1 代码提交流程

```
用户 → 前端 → API Gateway → 提交服务 → 判题调度 → 消息队列
                                ↓
                          数据库(pending)
                                
消息队列 → 判题节点 → 执行判题 → 更新结果 → 数据库
            ↓
        WebSocket/轮询 ← 前端 ← 用户
```

### 3.2 题目浏览流程

```
用户 → 前端 → API Gateway → 题目服务
                              ↓
                          检查 Redis 缓存
                          ├─ 命中 → 返回
                          └─ 未命中 → 查询数据库 → 写入缓存 → 返回
```

## 4. 安全设计

### 4.1 沙箱隔离
- Docker 容器隔离
- 禁止网络访问
- 限制系统调用
- 文件系统只读（除工作目录）

### 4.2 资源限制
- CPU 时间限制
- 内存使用限制
- 进程数限制
- 文件大小限制

### 4.3 代码安全
- 危险函数检测
- 恶意代码过滤
- 代码长度限制

## 5. 性能优化

### 5.1 缓存策略
- 题目列表缓存（5分钟）
- 题目详情缓存（10分钟）
- 用户信息缓存（30分钟）
- 排行榜缓存（1分钟）

### 5.2 数据库优化
- 索引优化
- 查询优化
- 连接池
- 读写分离（可选）

### 5.3 判题优化
- 多节点并行
- 预编译环境镜像
- 测试数据预加载
