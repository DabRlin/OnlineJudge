# Online Judge 系统 - 项目概述

## 1. 项目简介

Online Judge（在线判题系统）是一个用于编程竞赛、算法学习和编程能力评估的在线平台。用户可以提交代码解决问题，系统自动编译、运行并判定结果。

## 2. 核心功能

### 2.1 用户模块
- 用户注册、登录、权限管理
- 用户信息管理（个人资料、统计数据）
- 用户角色：普通用户、管理员、出题人

### 2.2 题目模块
- 题目浏览、搜索、分类
- 题目详情展示（描述、输入输出、样例、限制）
- 题目难度标签、标签分类
- 题目提交记录和通过率统计

### 2.3 判题模块（核心）
- 代码提交和排队
- 多语言支持（C/C++, Python, Java, Go 等）
- 代码编译
- 沙箱环境执行
- 测试数据对比
- 判题结果返回（AC, WA, TLE, MLE, RE, CE 等）

### 2.4 竞赛模块
- 竞赛创建和管理
- 实时排行榜
- 竞赛题目集
- 竞赛时间控制

### 2.5 讨论模块
- 题解分享
- 问题讨论
- 评论系统

## 3. 技术栈规划

### 3.1 前端
- **框架**：React + TypeScript
- **构建工具**：Vite
- **包管理器**：**bun** ⚡（极速安装，比 npm 快 20-30 倍）
- **运行时**：bun（开发环境）/ Node.js（生产环境可选）
- **UI 库**：Ant Design + TailwindCSS
- **状态管理**：Zustand / Redux Toolkit
- **代码编辑器**：Monaco Editor
- **数据请求**：@tanstack/react-query + Axios

### 3.2 后端
- **语言**：Python（推荐）
- **包管理器**：**uv** ⚡（极速安装，比 pip 快 10-100 倍）
- **框架**：Django + Django REST Framework
- **API**：RESTful API
- **认证**：JWT (PyJWT)
- **异步任务**：Celery
- **代码质量**：Ruff + Black + mypy

### 3.3 判题系统
- **沙箱**：Docker 容器隔离（namespace + cgroup + seccomp）
- **消息队列**：Redis（判题队列）
- **资源限制**：cgroup v2
- **安全加固**：AppArmor + Seccomp Profile
- **语言支持**：C/C++, Python, Java, Go, JavaScript (bun/Node.js)

### 3.4 数据库
- **关系型数据库**：PostgreSQL
- **缓存**：Redis
- **文件存储**：本地存储 / MinIO / OSS

### 3.5 开发工具链
- **Python 环境**：uv（虚拟环境 + 包管理）
- **JavaScript 环境**：bun（包管理 + 运行时 + 测试）
- **代码格式化**：Black (Python) + Prettier (JavaScript)
- **代码检查**：Ruff (Python) + ESLint (JavaScript)
- **类型检查**：mypy (Python) + TypeScript

### 3.6 部署
- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx
- **CI/CD**：GitHub Actions（使用 uv 和 bun 加速）
- **监控**：Prometheus + Grafana
- **日志**：ELK Stack / Loki

## 4. 系统特性

### 4.1 安全性
- 代码沙箱隔离
- SQL 注入防护
- XSS 防护
- CSRF 防护
- 权限控制

### 4.2 性能
- 判题队列异步处理
- 数据库查询优化
- Redis 缓存热点数据
- CDN 加速静态资源

### 4.3 可扩展性
- 微服务架构（可选）
- 判题节点水平扩展
- 数据库读写分离

## 5. 开发阶段

### Phase 1: MVP（最小可行产品）
- 基础用户系统
- 题目 CRUD
- 简单判题功能（支持 1-2 种语言）
- 基础前端界面

### Phase 2: 功能完善
- 多语言支持
- 竞赛系统
- 讨论区
- 管理后台

### Phase 3: 优化与扩展
- 性能优化
- 分布式判题
- 数据分析
- 社交功能

## 6. 项目目标

- 支持至少 5 种主流编程语言
- 单判题节点 QPS >= 10
- 99.9% 系统可用性
- 响应时间 < 200ms（非判题请求）
- 支持 10000+ 并发用户
