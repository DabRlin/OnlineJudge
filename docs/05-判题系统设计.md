# Online Judge - 判题系统设计

> 最后更新：2026-03-17

## 1. 概述

判题系统是 OJ 的核心模块，基于 **Docker + Celery** 实现异步、隔离的代码执行与结果判定。

代码路径：`src/judger/`

## 2. 整体流程

```
用户提交代码（POST /api/v1/submissions）
    │
    ▼
FastAPI: 创建 Submission 记录（status=pending）
    │ DockerJudgeService.submit_judge_task(submission_id)
    ▼
Celery 任务入队（Redis broker，队列名 "judge"）
    │
    ▼
Celery Worker（src/judger/tasks.py）消费任务
    │ 从数据库读取 Submission + Problem + TestCases
    ▼
DockerJudger.judge(code, language, test_cases, time_limit, memory_limit)
    │
    ├─ 逐个测试用例：创建 Docker 容器 → 执行 → 比对输出
    │
    ▼
更新 Submission 记录（status, score, time_used, memory_used, test_cases_result）
```

## 3. DockerJudger 实现

### 3.1 初始化（`docker_judger.py`）

```python
class DockerJudger:
    def __init__(self):
        # 优先使用 OrbStack socket（macOS）
        orbstack_socket = "~/.orbstack/run/docker.sock"
        if os.path.exists(orbstack_socket):
            self.client = docker.DockerClient(base_url=f"unix://{orbstack_socket}")
        else:
            self.client = docker.from_env()
```

### 3.2 判题入口

```python
def judge(
    self,
    code: str,
    language: Language,
    test_cases: list[dict],
    time_limit: int = 1000,    # 毫秒
    memory_limit: int = 256,   # MB
) -> JudgeResult
```

### 3.3 单测试点执行流程

1. 创建临时目录，写入源代码
2. 若需编译（C++/Java/Go），先在容器内编译
   - 编译失败 → 整题返回 `compile_error`
3. 逐测试用例：
   - `docker run` 启动容器，`stdin` 注入输入数据
   - 超时通过 `timeout` 参数控制，超时后 kill 容器
   - 比对容器 stdout 与期望输出（去除首尾空白）
   - 记录每个测试点状态

### 3.4 判题状态

| 状态 | 说明 |
|------|------|
| `accepted` | 全部通过 |
| `wrong_answer` | 输出不匹配 |
| `time_limit_exceeded` | 执行超时 |
| `memory_limit_exceeded` | 内存超限 |
| `runtime_error` | 运行时崩溃（非零退出） |
| `compile_error` | 编译失败 |
| `system_error` | Docker 异常等系统错误 |

## 5. Docker 容器安全配置

```python
container = client.containers.run(
    image=config.image,
    command=run_command,
    stdin_open=True,
    network_mode="none",          # 禁止所有网络访问
    mem_limit=f"{memory_limit}m", # 内存上限
    cpu_period=100000,
    cpu_quota=50000,              # CPU 使用率 50%
    read_only=True,               # 根文件系统只读
    tmpfs={"/tmp": "size=64m"},   # 临时可写目录
    remove=True,                  # 执行完自动删除
    detach=False,
    timeout=time_limit / 1000 + 1,
)
```

## 6. Celery 配置（`celery_app.py`）

```python
app = Celery("judger")
app.conf.update(
    broker_url="redis://localhost:6379/0",
    result_backend="redis://localhost:6379/0",
    task_queues=[
        Queue("judge"),    # 判题专用队列
        Queue("celery"),   # 默认队列
    ],
    task_serializer="json",
    result_expires=3600,
    worker_concurrency=2,  # 同时最多 2 个判题任务
)
```

启动命令：
```bash
cd src/judger
celery -A celery_app worker -Q judge,celery -c 2 --loglevel=info
```
