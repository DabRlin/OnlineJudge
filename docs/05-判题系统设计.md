# Online Judge 系统 - 判题系统设计

## 1. 判题系统概述

判题系统是 OJ 的核心模块，负责编译、运行用户提交的代码，并与标准答案进行对比，最终给出判题结果。

### 1.1 判题结果类型

| 状态码 | 英文缩写 | 中文名称 | 说明 |
|--------|---------|---------|------|
| AC | Accepted | 答案正确 | 程序输出完全正确 |
| WA | Wrong Answer | 答案错误 | 程序输出与标准答案不符 |
| TLE | Time Limit Exceeded | 超时 | 程序运行时间超过限制 |
| MLE | Memory Limit Exceeded | 内存超限 | 程序使用内存超过限制 |
| RE | Runtime Error | 运行错误 | 程序运行时崩溃 |
| CE | Compilation Error | 编译错误 | 程序编译失败 |
| PE | Presentation Error | 格式错误 | 输出格式不正确（可选） |
| OLE | Output Limit Exceeded | 输出超限 | 输出内容过多 |
| SE | System Error | 系统错误 | 判题系统内部错误 |

## 2. 判题流程设计

### 2.1 完整判题流程

```
┌─────────────────────────────────────────────────────────┐
│ 1. 接收提交                                              │
│    - 验证用户权限                                        │
│    - 验证题目存在                                        │
│    - 验证语言支持                                        │
│    - 验证代码长度                                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 2. 创建提交记录                                          │
│    - 生成提交 ID                                         │
│    - 保存到数据库（状态: pending）                       │
│    - 返回提交 ID 给用户                                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 3. 推入判题队列                                          │
│    - 根据优先级选择队列（竞赛 > 普通）                   │
│    - 将任务推入 Redis 队列                               │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 4. 判题节点获取任务                                      │
│    - Worker 从队列拉取任务                               │
│    - 更新状态为 judging                                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 5. 准备判题环境                                          │
│    - 创建临时工作目录                                    │
│    - 写入用户代码                                        │
│    - 获取测试数据                                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 6. 编译代码（如需要）                                    │
│    - 创建编译容器                                        │
│    - 执行编译命令                                        │
│    - 检查编译结果                                        │
│      ├─ 成功 → 继续                                     │
│      └─ 失败 → 返回 CE，结束判题                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 7. 运行测试点                                            │
│    For each test case:                                  │
│    ├─ 创建运行容器                                      │
│    ├─ 设置资源限制（CPU、内存、时间）                   │
│    ├─ 挂载代码和测试数据                                │
│    ├─ 执行程序                                          │
│    ├─ 捕获输出和资源使用                                │
│    ├─ 对比输出                                          │
│    │  ├─ AC → 继续下一个测试点                         │
│    │  ├─ WA → 记录错误，继续或停止                     │
│    │  ├─ TLE → 记录超时，停止                          │
│    │  ├─ MLE → 记录超内存，停止                        │
│    │  └─ RE → 记录运行错误，停止                       │
│    └─ 销毁容器                                          │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 8. 汇总结果                                              │
│    - 计算总分                                            │
│    - 统计通过测试点数                                    │
│    - 记录最大时间和内存                                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 9. 更新数据库                                            │
│    - 更新提交状态                                        │
│    - 保存判题结果                                        │
│    - 更新用户统计                                        │
│    - 更新题目统计                                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 10. 清理环境                                             │
│    - 删除临时文件                                        │
│    - 清理容器                                            │
│    - 释放资源                                            │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 11. 通知用户                                             │
│    - WebSocket 推送结果                                  │
│    - 或等待用户轮询                                      │
└─────────────────────────────────────────────────────────┘
```

## 3. 沙箱设计

### 3.1 Docker 沙箱方案

使用 Docker 容器作为沙箱环境，提供进程隔离和资源限制。

**优点：**
- 完善的隔离性
- 易于配置和管理
- 支持多种语言环境
- 资源限制精确

**配置示例：**

```python
docker_config = {
    'image': 'judge/cpp:latest',
    'network_disabled': True,  # 禁用网络
    'mem_limit': '256m',       # 内存限制
    'memswap_limit': '256m',   # 禁用 swap
    'cpu_period': 100000,
    'cpu_quota': 100000,       # CPU 限制 100%
    'pids_limit': 50,          # 进程数限制
    'read_only': True,         # 只读文件系统
    'volumes': {
        '/tmp/judge/code': {
            'bind': '/workspace',
            'mode': 'rw'
        }
    },
    'security_opt': ['no-new-privileges'],
    'cap_drop': ['ALL'],       # 移除所有能力
}
```

### 3.2 资源限制

#### 3.2.1 时间限制

- **CPU 时间**: 使用 cgroup 限制 CPU 使用
- **实际时间**: 使用 timeout 命令或程序内计时
- **时间测量**: 精确到毫秒

```python
# 时间限制实现
import time
import signal

def timeout_handler(signum, frame):
    raise TimeoutError("Time Limit Exceeded")

signal.signal(signal.SIGALRM, timeout_handler)
signal.alarm(time_limit_seconds)

try:
    # 执行程序
    result = run_program()
except TimeoutError:
    return "TLE"
finally:
    signal.alarm(0)
```

#### 3.2.2 内存限制

- 使用 Docker 的 `--memory` 参数
- 监控容器内存使用
- 超限时立即终止

#### 3.2.3 其他限制

- **输出大小**: 限制为 64MB
- **进程数**: 限制为 1-50 个
- **文件大小**: 限制为 10MB
- **系统调用**: 使用 seccomp 过滤

## 5. 输出对比策略

### 5.1 标准对比（默认）

- 逐行对比
- 忽略行尾空格
- 忽略文件尾空行

```python
def standard_compare(user_output, expected_output):
    user_lines = user_output.strip().split('\n')
    expected_lines = expected_output.strip().split('\n')
    
    if len(user_lines) != len(expected_lines):
        return False
    
    for user_line, expected_line in zip(user_lines, expected_lines):
        if user_line.rstrip() != expected_line.rstrip():
            return False
    
    return True
```

### 5.2 严格对比

- 完全一致，包括空格和换行

```python
def strict_compare(user_output, expected_output):
    return user_output == expected_output
```

### 5.3 浮点数对比

- 支持相对误差和绝对误差

```python
def float_compare(user_output, expected_output, eps=1e-6):
    user_nums = extract_numbers(user_output)
    expected_nums = extract_numbers(expected_output)
    
    if len(user_nums) != len(expected_nums):
        return False
    
    for u, e in zip(user_nums, expected_nums):
        if abs(u - e) > eps and abs(u - e) / max(abs(e), 1) > eps:
            return False
    
    return True
```

### 5.4 Special Judge

对于有多个正确答案的题目，使用自定义判题程序。

```python
def special_judge(input_file, user_output, expected_output):
    # 运行 SPJ 程序
    result = run_spj_program(
        spj_exe='./spj',
        input_file=input_file,
        user_output=user_output,
        expected_output=expected_output
    )
    return result  # 0: AC, 1: WA
```

## 6. 判题队列设计

### 6.1 多优先级队列

使用 Redis 实现三个优先级队列：

```python
QUEUE_CONFIG = {
    'high': 'judge:queue:high',      # 竞赛提交
    'normal': 'judge:queue:normal',  # 普通提交
    'low': 'judge:queue:low',        # 重判任务
}
```

### 6.2 任务分发策略

```python
def get_next_task():
    # 优先处理高优先级队列
    task = redis.rpop('judge:queue:high')
    if task:
        return task
    
    # 然后处理普通队列
    task = redis.rpop('judge:queue:normal')
    if task:
        return task
    
    # 最后处理低优先级队列
    task = redis.rpop('judge:queue:low')
    return task
```

### 6.3 任务重试机制

```python
def handle_task_failure(task, error):
    retry_count = task.get('retry_count', 0)
```
